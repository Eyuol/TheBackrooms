# 逃离后室 设计文档

## 游戏背景

《逃离后室》是一款后室主题的同步路径解谜游戏。玩家操控克拉克在异常空间中寻找出口。每个房间的 Entrance 会读取克拉克刚进入房间后的短暂行动，并把这段路径刻成一段“回声”。记录归零后，【影子】海盗克拉克从入口出现，像录像一样严格播放克拉克前半段的每一步。

影子不会一直跟随克拉克，因为它不是实时追踪者，而是后室入口生成的短暂路径回声。它只能播放被记录下来的那几步；步数耗尽后，回声能量耗尽，就会停在最后一格。玩家要利用这个停下来的影子压住按钮，为克拉克打开或关闭激光平台。

克拉克的目标很简单：到达出口并逃离。难点在于影子会复刻路径，玩家不能与影子同格相撞，也不能掉入没有落点的断层。部分 Exit 或伸缩平台需要按钮保持按下，通常需要让影子最终停在按钮上，为克拉克打开通路。

## 核心玩法

1. 记录阶段：克拉克先行动固定步数。顶部倒计时显示“距离【影子】海盗克拉克出现还有 x 步”。
2. 影子出现：倒计时归零后，影子从 Entrance 出现，开始逐步复刻记录阶段的方向；它只会走完被记录的步数。
3. 同步逃离：克拉克继续行动，影子同步执行旧路线。二者不能同格，也不能交叉对撞。
4. 机关协作：按钮会改变对应激光平台状态。影子停在按钮上时，通路会保持当前机关状态。
5. 通关条件：克拉克到达出口。
6. 失败条件：克拉克或影子掉入没有落点的断层、影子断步、克拉克与影子碰面。

## 地图组件

地图由前景组件拼接，而不是把背景当地图。背景只是后室环境，真正可走区域由下列组件组成：

- Entrance：入口。克拉克初始位置，也是影子出现的位置。地图数据中用 `S` 标记。
- Exit：出口。克拉克的目标。地图数据中用 `G` 标记；可通过 `exitGate` 绑定按钮锁。
- Platform：普通平台。地图数据中用 `floor` 线段生成。
- Ladder：竖梯。地图数据中用 `stair` 线段生成；上下楼梯计为一步。
- Button：压力按钮。小写 `a/b/c/d` 表示按钮，分别控制 `A/B/C/D`。
- Retractable Platform：伸缩平台。大写 `A/B/C/D` 表示受控平台；对应按钮未按下时收回，按下后变成可走平台。
- Gap / Drop：断层。横向走到空格时，如果同列下方存在可走平台，会直接下落到最近平台；没有落点则失败。

3 步之后，【影子】海盗克拉克将从入口出现。

地图描述：第一层是一条横向长平台，有六个平台，分别是（伸缩平台，普通平台，带按钮的平台，普通平台，普通平台，入口），最左侧是伸缩平台，常态是关闭（类似普通平台，掉不下去），按钮按下是打开（会掉下去）。第二层有五个平台，除了最右侧是出口，其他都是普通平台。

设计理念：让玩家理解最基础的“记录路径 -> 影子复刻 -> 影子停按钮 -> 克拉克进出口”。这一关路线短，重点是展示影子并不是单纯的敌人，而是需要规划利用的同步对象。

## 当前实现说明

- 地图数据位于 `index.html` 的 `LEVELS` 数组。
- 可走组件由 `segments` 生成，水平 `floor` 是平台，竖直 `stair` 是梯子。
- 机关由 `marks` 生成，小写是按钮，大写是伸缩平台。
- `exitGate` 用来声明 Exit 需要哪个按钮开启。
- 影子不会自由寻路，只会严格执行记录阶段的方向序列。
- 每次行动后顶部会显示当前已走路线，方便玩家核对步骤。
